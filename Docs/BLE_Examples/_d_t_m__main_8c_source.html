266 /*
267  ******************************************************************************
268  ******************************************************************************
269  * The DTM project memory layout is as follows:
270  *
271  * ------------- 0x10067FFF
272  *
273  * DTM
274  *
275  * ------------- 0x10042000 (MEMORY_FLASH_APP_OFFSET)
276  * DTM Updater
277  * ------------- 0x10040000 (_MEMORY_FLASH_BEGIN_)
278  *
279  * The DTM UART/UART_Sleep, SPI project configurations include the DTM updater code +
280  * the DTM specific code (Configuration --> Preprocessor option: UART --> UART_INTERFACE; UART Sleep --> UART_SLEEP; SPI --> SPI_INTERFACE).
281  * The DTM SPI_NOUPDATER, UART_NOUPDATER project configurations allows to build the binary
282  * images (with based address = MEMORY_FLASH_APP_OFFSET) which includes only the DTM specific code. (Configuration --> Preprocessor option: UART_NOUPDATER --> UART_INTERFACE; SPI_NOUPDATER --> SPI_INTERFACE).
283  * This binary image can be used for upgrading the DTM binary image on a device:
284  * the assumption is that the device has been previosly loaded with the DTM full image (DTM + DTM Updater built with DTM UART/UART_Sleep, SPI project configurations),
285  * and that the user just wants to upgrade the DTM specific code with a new version, using the DTM SPI_NOUPDATER, UART_NOUPDATER project configurations
286  * binary images.
287  * The DTM Updater allows to access the memory flash through ACI_HAL commands.
288  * It is placed on the bootloader section from _MEMORY_FLASH_BEGIN_ to
289  * _MEMORY_FLASH_BEGIN_ + MEMORY_FLASH_APP_OFFSET (0x2000).
290  * The DTM Updater can be activated in the following way:
291  * 1) Activation by using ACI_HAL_UPDATER_START
292  * 2) Activation by using IO3 pin (high level at start up).
293  * Note: if the IO3 pin is used and is high at start up, this will cause the
294  * DTM Updater starts. So, to avoid this, the support of the DTM Updater
295  * can be removed.
296  *
297  * If the DTM Updater support is not necessary, the DTM Updater code can be
298  * removed:
299  * 1) Disable/Remove the file DTM_Updater_xxx.c inside the folder project
300  * DTM_Updater.
301  * 2) Option/Linker: Change the symbol definition from
302  * MEMORY_FLASH_APP_OFFSET=0x2000 to MEMORY_FLASH_APP_OFFSET=0.
303  *
304  * The same communication port (SPI or UART with related pins) of the DTM
305  * is used for the DTM Updater. Any change of relevant pins must be reported
306  * also in DTM Updater firmware.
307  *
308  * NOTES: UART default baudrate is 115200;
309  *
310  * UART_Throughput configuration must be used only for throughput evaluation tests
311  * through the BlueNRG GUI, Throughput tab.
312  *
313  ******************************************************************************
314  ** WARNING **
315  ******************************************************************************
316  * DTM v.3.2 available on BlueNRG-1,2 SDK v. 3.2.0 has an offset of 0x2000 (8 KB).
317  * This image is not compatible with DTM_Updater image provided on previous SDK
318  * versions (offset = 0x800).
319  * In order to make it compatible with previous DTM_Updater versions, please
320  * rebuild the DTM UART or SPI configurations with linker symbol
321  * MEMORY_FLASH_APP_OFFSET=0x800.
322  ******************************************************************************
323  ******************************************************************************
324 */
