Detailed Description

This application contains an example which shows how implementing the Sensor Demo application tailored for interacting with ST BLE Sensor smartphone app (previously known as ST BlueMS). The device sends periodically, to the BlueMS APP, the data collected from the motion sensors: accelerometer and gyroscope. And from the environmental sensors: pressure sensor and temperature sensor. The usage is similar to the firmware example BLE_Examples/SensorDemo.

ATOLLIC project

To use the project with ATOLLIC TrueSTUDIO for ARM, please follow the instructions below:

  1. Open the ATOLLIC TrueSTUDIO for ARM and select File->Import... Project menu.
  2. Select Existing Projects into Workspace.
  3. Select the ATOLLIC project
  4. Select desired configuration to build from Project->Manage Configurations
  5. Select Project->Rebuild Project. This will recompile and link the entire application
  6. To download the binary image, please connect STLink to JTAG connector in your board (if available).
  7. Select Project->Download to download the related binary image.
  8. Alternatively, open the BlueNRG1 Flasher utility and download the built binary image.

KEIL project

To use the project with KEIL uVision 5 for ARM, please follow the instructions below:

  1. Open the KEIL uVision 5 for ARM and select Project->Open Project menu.
  2. Open the KEIL project C:\{username}-1_2 DK x.x.x\Project\BLE_Examples\BLE_SensorDemo_BlueMSapp\MDK-ARM\BlueNRG-1\BLE_SensorDemo_BlueMSapp.uvprojx or C:\{username}-1_2 DK x.x.x\Project\BLE_Examples\BLE_SensorDemo_BlueMSapp\MDK-ARM\BlueNRG-2\BLE_SensorDemo_BlueMSapp.uvprojx
  3. Select desired configuration to build
  4. Select Project->Rebuild all target files. This will recompile and link the entire application
  5. To download the binary image, please connect STLink to JTAG connector in your board (if available).
  6. Select Project->Download to download the related binary image.
  7. Alternatively, open the BlueNRG1 Flasher utility and download the built binary image.

IAR project

To use the project with IAR Embedded Workbench for ARM, please follow the instructions below:

  1. Open the Embedded Workbench for ARM and select File->Open->Workspace menu.
  2. Open the IAR project C:\{username}-1_2 DK x.x.x\Project\BLE_Examples\BLE_SensorDemo_BlueMSapp\EWARM\BlueNRG-1\BLE_SensorDemo_BlueMSapp.eww or C:\{username}-1_2 DK x.x.x\Project\BLE_Examples\BLE_SensorDemo_BlueMSapp\EWARM\BlueNRG-2\BLE_SensorDemo_BlueMSapp.eww
  3. Select desired configuration to build
  4. Select Project->Rebuild All. This will recompile and link the entire application
  5. To download the binary image, please connect STLink to JTAG connector in your board (if available).
  6. Select Project->Download and Debug to download the related binary image.
  7. Alternatively, open the BlueNRG1 Flasher utility and download the built binary image.

Usage

This is a demonstration example of the Sensor Demo application version tailored for interacting with the ST BlueMS smarthphone application. Refer to the related documentation for more details.

This is a demonstration example of the BLE Sensor Demo application version tailored for interacting with the ST BLE Sensor smarthphone application (previously known as ST BlueMS). This demo application allows to configure a BlueNRG-1/2 kit platform with a demo application able to interact with the ST BLE Sensor application and provide it a set of sensor data, which user can log to different cloud providers.

NOTEs:

  • OTA service support for lower or higher application is enabled, respectively, through ST_OTA_LOWER_APPLICATION=1 or ST_OTA_HIGHER_APPLICATION=1 (preprocessor, linker) options and files: OTA_btl.[ch] (refer to LowerApp_OTA and HigherApp_OTA IAR workspaces).
  • OTA service manager support is enabled, respectively, through ST_USE_OTA_SERVICE_MANAGER_APPLICATION (preprocessor, linker) options and files: OTA_btl.[ch] (refer to Use_OTA_ServiceManager IAR workspace).
