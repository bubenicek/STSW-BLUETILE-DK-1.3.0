Detailed Description

This code provides an example how to configure and to use the I2C master.

ATOLLIC project

To use the project with ATOLLIC TrueSTUDIO for ARM, please follow the instructions below:

  1. Open the ATOLLIC TrueSTUDIO for ARM and select File->Import... Project menu.
  2. Select Existing Projects into Workspace.
  3. Select the ATOLLIC project
  4. Select desired configuration to build from Project->Manage Configurations
  5. Select Project->Rebuild Project. This will recompile and link the entire application
  6. To download the binary image, please connect STLink to JTAG connector in your board (if available).
  7. Select Project->Download to download the related binary image.
  8. Alternatively, open the BlueNRG1 Flasher utility and download the built binary image.

KEIL project

To use the project with KEIL uVision 5 for ARM, please follow the instructions below:

  1. Open the KEIL uVision 5 for ARM and select Project->Open Project menu.
  2. Open the KEIL project C:\{username}-1_2 DK x.x.x\Project\BlueNRG1_Periph_Examples\I2C\Master_Polling\MDK-ARM\{BlueNRG-1-2}\I2C_Master_Polling.uvprojx
  3. Select desired configuration to build
  4. Select Project->Rebuild all target files. This will recompile and link the entire application
  5. To download the binary image, please connect STLink to JTAG connector in your board (if available).
  6. Select Project->Download to download the related binary image.
  7. Alternatively, open the BlueNRG1 Flasher utility and download the built binary image.

IAR project

To use the project with IAR Embedded Workbench for ARM, please follow the instructions below:

  1. Open the Embedded Workbench for ARM and select File->Open->Workspace menu.
  2. Open the IAR project C:\{username}-1_2 DK x.x.x\Project\BlueNRG1_Periph_Examples\I2C\Master_Polling\EWARM\{BlueNRG-1-2}\I2C_Master_Polling.eww
  3. Select desired configuration to build
  4. Select Project->Rebuild All. This will recompile and link the entire application
  5. To download the binary image, please connect STLink to JTAG connector in your board (if available).
  6. Select Project->Download and Debug to download the related binary image.
  7. Alternatively, open the BlueNRG1 Flasher utility and download the built binary image.

Usage

The I2C controller is an interface designed to support the physical and data link layer according to the I2C standard revision 3.0 (June 2007). The I2C bus is a 2-wire serial bus that provides a low-cost interconnection between ICs.

This example provides a simple application where the I2C communication is controlled by polling of the I2C status register content.

In this example, two boards should be used: the Master board is the board with the Master_Polling firmware code, while the Slave board is the board with the Slave_Polling firmware code. The Master board shows a small command line interface through UART (USB-to-SERIAL must be connected to the PC). The user can change the LEDs status of the Slave board by using the command line interface commands. In addition, the information about the status of the LEDS on the Slave board is read. I2C is used to transfer the information and change the status of the LEDs on the slave board.

The I2C is configured in the master mode. The I2C clock frequency is set to 10 kHz.

The two boards must be connected by wire according to the below scheme.
