Detailed Description

Code demostrating a TX/RX scenario where a payload greater than 32 bytes is exchanged. This code implements the receiver side. Three devices are necessary to run fully this demo.

ATOLLIC project

To use the project with ATOLLIC TrueSTUDIO for ARM, please follow the instructions below:

  1. Open the ATOLLIC TrueSTUDIO for ARM and select File->Import... Project menu.
  2. Select Existing Projects into Workspace.
  3. Select the ATOLLIC project
  4. Select desired configuration to build from Project->Manage Configurations
  5. Select Project->Rebuild Project. This will recompile and link the entire application
  6. To download the binary image, please connect STLink to JTAG connector in your board (if available).
  7. Select Project->Download to download the related binary image.
  8. Alternatively, open the BlueNRG1 Flasher utility and download the built binary image.

  7. Alternatively, open the BlueNRG1 Flasher utility and download the built binary image.

Usage

This code is a component of the TX/RX scenario where a payload greater than 32 bytes is exchanged. This code implements the receiver side. Program 2 devices with the RX configuration (2 differents access addresses) and one device with the TX configuration. The communication requires ACK to confirm the packet reception. In this example the ActionPacket mechanism exported by the BlueNRG_radio driver is used to schedule the reception of the entire payload and then send the ACK. The ActionPacket mechanism allows a much flexible use of the internal state machine. In this case, three ActionPackets are used: the first two ActionPackets manage the reception, while the last one is used to schedule the transmission of the ACK packet (1 byte of payload equals to 0xAE). The conditionRoutine is used to handle the state machine defined by the user. The code for RX should be used to program two boards. The user must use the symbol ACCESS_ADDRESS to program the two boards with different access address. In this way, the transmitter can communicate with two RX boards.

If TX is communicating with both RX devices (configured from user with 2 different ACCESS_ADDRESS), on TX serial terminal, the following information should appear:

  • PER = 0.0% PCKT sent 1000, ACK received 1000 Retry 0
  • PER = 0.0% PCKT sent 1000, ACK received 1000 Retry 0

If TX is communicating with only one RX device (second RX device not present), on TX serial terminal, the following information should appear:

  • PER = 0.0% PCKT sent 1000, ACK received 1000 Retry 0
  • PER = 100.0% PCKT sent 1000, ACK received 0 Retry 3000

If no RX device is present, on TX serial terminal, the following information should appear:

  • PER = 100.0% PCKT sent 1000, ACK received 0 Retry 3000
  • PER = 100.0% PCKT sent 1000, ACK received 0 Retry 3000
