Detailed Description

Code demostrating a TX/RX scenario where a payload greater than 32 bytes is exchanged. This code implements the transmitter side. Two devices are necessary to run fully this demo.

ATOLLIC project

To use the project with ATOLLIC TrueSTUDIO for ARM, please follow the instructions below:

  1. Open the ATOLLIC TrueSTUDIO for ARM and select File->Import... Project menu.
  2. Select Existing Projects into Workspace.
  3. Select the ATOLLIC project
  4. Select desired configuration to build from Project->Manage Configurations
  5. Select Project->Rebuild Project. This will recompile and link the entire application
  6. To download the binary image, please connect STLink to JTAG connector in your board (if available).
  7. Select Project->Download to download the related binary image.
  8. Alternatively, open the BlueNRG1 Flasher utility and download the built binary image.

Usage

This code is a component of the TX/RX scenario where a payload greater than 32 bytes is exchanged.. This code implements the transmitter side: program one device with the TX configuration and the other two with the RX configuration (with 2 differents access addresses). The transmitter sends a number of packets equals to MAX_NUM_PACKET.The communication requires ACK to confirm the packet reception. The transmitter will retry up to 3 times in order to have a good communication. If the ACK packet is not received, the same packet is sent for retry. In this example the hal_radio layer is used in order to schedule the transmissions. The callback TXCallback provided to the API HAL_RADIO_SendPacketWithAck define the behavior of the radio according to the IRQ flags. Once the packet has been correctly received, the transmitter selects a new access address to communicate with the other receiver. And it will continue in this way, sending packet alternately to the two receivers. Once the maximum number of packets to send is reached, the summary of the communication is printed out with the information on the timeout errors, CRC errors, and the Packet Error Rate (PER).

If TX is communicating both RX devices (configured from user with 2 different ACCESS_ADDRESS), on TX serial terminal, the following information should appear:

  • PER = 0.0% PCKT sent 1000, ACK received 1000 Retry 0
  • PER = 0.0% PCKT sent 1000, ACK received 1000 Retry 0

If TX is communicating with only one RX device (second RX device not present), on TX serial terminal, the following information should appear:

  • PER = 0.0% PCKT sent 1000, ACK received 1000 Retry 0
  • PER = 100.0% PCKT sent 1000, ACK received 0 Retry 3000

If no RX device is present, on TX serial terminal, the following information should appear:

  • PER = 100.0% PCKT sent 1000, ACK received 0 Retry 3000
  • PER = 100.0% PCKT sent 1000, ACK received 0 Retry 3000
