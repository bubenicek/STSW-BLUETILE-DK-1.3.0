Detailed Description

Code demostrating a simple TX/RX scenario with sleep management. This code implements the receiver side. Two devices are necessary to run fully this demo.The RX_vtimer configuration also allows to show how to use a virtual timer (sleep timer) in order to toggle a led every 2 seconds.

ATOLLIC project

To use the project with ATOLLIC TrueSTUDIO for ARM, please follow the instructions below:

  1. Open the ATOLLIC TrueSTUDIO for ARM and select File->Import... Project menu.
  2. Select Existing Projects into Workspace.
  3. Select the ATOLLIC project
  4. Select desired configuration to build from Project->Manage Configurations
  5. Select Project->Rebuild Project. This will recompile and link the entire application
  6. To download the binary image, please connect STLink to JTAG connector in your board (if available).
  7. Select Project->Download to download the related binary image.
  8. Alternatively, open the BlueNRG1 Flasher utility and download the built binary image.

Usage

Code demostrating a simple TX/RX scenario. This code implements the receiver side. Program two devices: one with the TX configuration and the other with the RX configuration. On the RX configuration, open a serial terminal to get the packets received. The communication requires ACK to confirm the packet reception. In this example the hal_radio layer is used in order to schedule the reception. The callback RxCallback provided to the API HAL_RADIO_ReceivePacketWithAck define the behavior of the radio according to the IRQ flags. Once a packet is received, the received data are printed out on a serial terminal. The callback App_SleepMode_Check is used to avoid the sleep during the UART communication. The RX_vtimer configuration also allows to show how to use a virtual timer (sleep timer) in order to toggle a led every 2 seconds.
