Detailed Description

This example demonstrates how to use the RTC timer for periodic interrupt generation with use of patterns.

ATOLLIC project

To use the project with ATOLLIC TrueSTUDIO for ARM, please follow the instructions below:

  1. Open the ATOLLIC TrueSTUDIO for ARM and select File->Import... Project menu.
  2. Select Existing Projects into Workspace.
  3. Select the ATOLLIC project
  4. Select desired configuration to build from Project->Manage Configurations
  5. Select Project->Rebuild Project. This will recompile and link the entire application
  6. To download the binary image, please connect STLink to JTAG connector in your board (if available).
  7. Select Project->Download to download the related binary image.
  8. Alternatively, open the BlueNRG1 Flasher utility and download the built binary image.

  4. Select Project->Rebuild all target files. This will recompile and link the entire application
  5. To download the binary image, please connect STLink to JTAG connector in your board (if available).

Usage

The BlueNRG-1,2 embed a real-time counter RTC clocked by the RC32K and can still run in low-power modes. The RTC provides to the user real-time clockwatch functionality and a real-time clock timer. The RTC clockwatch consists of two counters and two alarm registers that have the following features.

Two counters:

  • Counts seconds, minutes, hours, days of the week, days of the month.
  • Counts years.

Two alarm registers:

  • Can be used to trigger an interrupt or device wake-up at an exact date and time.

The RTC timer is a 32-bit free-running counter, clocked by the 32 kHz clock signal (from an embedded 32 kHz RC), that could work in periodic or single shot mode.

In this example, the periodic mode is used together with a pattern configuration. The RTC is configured in the periodic timer mode and the load value to registers RTC_TLR1 and RTC_TLR2 value is set. The RTC_TLR1 is set to generate time period of 1s, the RTC_TLR2 is set to generate time period of 100ms. Then the RTC is enabled. Each time the RTC timer reach value 0 it generates an interrupt and the timer value is automatically re-loaded from register RTC_TLR1 or RTC_TLR2 register depending on the pattern register setting.

The pattern size is set to 8 bits and the pattern is set to 0xF2 (the endianess is LSB) that generates the following pattern in bits 0b01001111. This means that the RTC generates: 1 period defined by RTC_TLR1 (1 s), 1 periods defined by RTC_TLR2 (100 ms), 2 periods defined by RTC_TLR1 (2 s) and 4 periods defined by RTC_TLR2 (400 ms). Once the pattern is finished it is repeated again.

Within the RTC interrupt routine the LED DL1 (IO6) is toggled.
