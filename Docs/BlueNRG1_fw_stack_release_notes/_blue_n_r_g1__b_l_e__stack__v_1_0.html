Bluetooth Low Energy (BLE) stack binary image files

This section lists the BLE stack binary library files to be used for the BlueNRG-1 System On Chip configuration:

  • BlueNRG-1 Bluetooth Low Energy (BLE) Stack binary libraries v1.0:
    • IAR, EWARM for ARM library: bluenrg1_stack_lib.a
    • Keil, MDK-ARM library: bluenrg1_stack_lib_keil.lib
    • Atollic, TrueStudio library: libbluenrg1_stack_lib_atollic.a
  • NOTES:
    • BlueNRG-1 Bluetooth Low Energy (BLE) stack binary library files v1.0 are available on folder: Library\Bluetooth_LE\library . Each file must be linked to the built user application in order to get access to the BlueNRG-1 BLE stack features.

Bluetooth Low Energy (BLE) stack network coprocessor configuration

  • Two network coprocessor modes are supported: SPI and UART (mainly for RF evaluation purpose).
  • The network coprocessor over SPI specification aims at achieving the following targets:
    • Very similar to BlueNRG, BlueNRG-MS protocol
    • Power efficient
    • Code efficient
    • Fast data transfer
  • DTM project and related workspaces provides reference appplications for configuring the BlueNRG-1 platform (STEVAL-IDB007V1) as a network coprocessor (refer to BlueNRG-1 DK Release notes documentation for more information).

Limitations/Bugs

This section lists the issues known at firmware image release time. Issues found after this release are listed as solved or limitations on newer releases.

  • ID 2057: Active bonding informations are removed from Security Database on DB full
  • ID 2025: Device doesn't go to sleep after long write procedure
  • ID 2011: Missing HCI_DISCONNECTION_COMPLETE_EVENT event
  • ID 1688: BlueNRG does not stop advertising immediately, but only after next scheduled packet.
  • ID 1687: Scanning performed with a maximum duty cycle of 95%
  • ID 1679: Direct connection establishment procedure cannot be cancelled after a connection has been established as a slave
  • ID 1676: Privacy-enabled slave allowed to use non-resolvable private address in connectable mode, while it has to be forbidden
  • ID 1675: A privacy-enabled Central is allowed to use static and public addresses during some GAP procedures
  • ID 1673: It is not possible to use whitelist with ACI_GAP_START_OBSERVATION_PROC
  • ID 1663: No service changed indication sent if a reset occurs before disconnection
  • ID 1661: Device may hang for 32s if XO start-up time exceedes the IFR HS_STARTUP_TIME value
  • ID 1659: Not able to do fast advertise when all the connections have long connection intervals
